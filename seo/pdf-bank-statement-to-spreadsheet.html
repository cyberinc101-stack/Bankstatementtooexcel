Why PDF Bank Statements Are Difficult to Work With

PDF was designed for printing, not data. Bank statement PDFs look great on screen but their data is locked inside — you can't sort, filter, or calculate in a PDF. Converting that data to a spreadsheet unlocks everything.

The challenge: bank PDFs don't have a standard structure. Chase formats their PDFs differently from Wells Fargo, which is different from ANZ, Barclays, or any of the thousands of banks worldwide. Traditional PDF parsers fail on most of them.

Why AI Works Where Traditional Parsers Fail

StatementExcel uses the Claude AI model to read your PDF the way a human would — understanding context, recognizing transaction patterns, handling multi-line descriptions, and detecting the currency and date format automatically. This is why accuracy is 95%+ versus 60–70% for rule-based PDF parsers.

How it works: Your PDF is encoded as base64 and sent directly to the Anthropic Claude API using your personal API key. The AI returns a structured JSON array of every transaction, which StatementExcel then formats into your chosen spreadsheet layout.

What Data Gets Extracted

For each transaction, StatementExcel extracts:

  • Date — normalized to YYYY-MM-DD format
  • Reference number — transaction ID, cheque number, or reference code if present
  • Description — full merchant name or transaction description
  • Currency — detected from the statement (USD, NZD, GBP, EUR, AUD, etc.)
  • Amount — negative for debits, positive for credits
  • Running balance — if shown in the statement
  • Type — debit or credit
  • Category — AI-guessed category (Groceries, Utilities, Salary, Dining, etc.)

Which PDF Bank Statements Work?

Text-based PDFs work — these are statements downloaded digitally from online banking portals. Scanned PDFs (photographed or printed-then-scanned paper statements) do not work, as there is no extractable text.

If you're unsure whether your PDF is text-based: open it in a PDF reader and try to select/highlight text. If you can highlight individual words, it's text-based and will work. If clicking selects the whole page as an image, it's a scanned PDF.
