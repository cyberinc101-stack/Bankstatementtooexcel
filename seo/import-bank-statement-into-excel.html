Method 1: Convert PDF to CSV First (Recommended)

Step 1: Download your statement as a PDF

Log into your bank's online banking portal. Navigate to Statements or Documents. Download your statement as a PDF (not a printed/scanned copy). Most banks label this "e-Statement" or "PDF Statement".

Step 2: Convert to CSV with StatementExcel

Upload your PDF to StatementExcel. Enter your Claude API key (free from console.anthropic.com). Click Convert. In 10–30 seconds you'll see a live preview of all your transactions.

Step 3: Choose your layout and download

Pick from four Excel layouts — Standard, Split Columns, Accountant, or Minimal — and customize which columns appear. Then click Download CSV.

Step 4: Open CSV in Excel

  1. Double-click the downloaded CSV file, or open Excel and go to File → Open
  2. If the Text Import Wizard appears: select Delimited → Comma → Finish
  3. Your transactions are now in a proper Excel table
  4. Select all data and press Ctrl+T to format as an Excel Table for easier filtering

Method 2: Excel Power Query (PDF Tables)

Excel's built-in Power Query can sometimes import PDF tables directly — but only if the PDF has a clean table structure. Most bank statement PDFs do not have clean tables, making this unreliable. If you want to try it:

  1. In Excel, go to Data → Get Data → From File → From PDF
  2. Select your bank statement PDF
  3. Power Query shows detected tables — select the transaction table
  4. Click Transform Data to clean up columns, then Load

This works for some statements but often requires significant cleanup. The AI converter is far more reliable.

Analyzing Your Bank Statement in Excel

Once imported, you can use Excel's built-in tools:

  • Data → Filter: Filter by category (Groceries, Dining, etc.) or amount range
  • Insert → PivotTable: Summarize total spending by category or month
  • Insert → Chart: Visualize spending patterns over time
  • Formulas: Use SUMIF to total by category, or AVERAGEIF for average transaction sizes
Pro tip: In Excel, select your transaction data and press Ctrl+T to create a formatted Table. This enables auto-filtering on every column and makes PivotTables much easier to build.

Common Problems When Importing Bank Statements to Excel

Amounts showing as text instead of numbers

Select the Amount column, go to Data → Text to Columns → Delimited → Finish. This forces Excel to re-parse the column as numbers.

Dates not recognized as dates

Select the Date column, right-click → Format Cells → Date, and choose your preferred date format. If dates still show as text, use the DATEVALUE() formula to convert them.

Characters showing as ??? or garbled

This is an encoding issue. When opening the CSV, make sure to select UTF-8 encoding in the Text Import Wizard. StatementExcel includes a BOM character that usually handles this automatically.

Remember: StatementExcel only works with text-based PDFs from online banking portals. Scanned paper statements are image-based and cannot be parsed. Check our FAQ for more.
